# AstraForge documentation

Welcome to the Nextra-powered docs site for AstraForge. This space explains how to run the platform locally, how the monorepo is organized, and how to contribute changes confidently.

## What you will find

- Quick setup steps for local development.
- Production deployment guidance (Compose or Kubernetes) plus configuration tips.
- Architecture overview and sandbox behavior.
- Sandbox Python client examples and operational runbooks.
- Guidance for extending the backend, frontend, and docs.

## Quick links

- [Getting started](/getting-started) walks through installing dependencies and running services.
- [Production deployment](/production) covers hardened settings, migrations, and image pulls.
- [Configuration & secrets](/configuration) documents the key environment variables.
- [Architecture](/architecture) summarizes the system design and points to the full diagrams.
- [Sandbox overview](/sandbox-overview) explains lifecycle, security, and APIs.
- [Sandbox Python client](/sandbox-python-client) shows how to drive the sandbox from Python.
- [Operations & testing](/operations) lists lint/test commands, reaper behavior, and health checks.
- [Contributing](/contributing) captures conventions for code and documentation updates.

## Using this site

The docs are built with [Nextra](https://nextra.site/) on Next.js and can be served locally with `pnpm dev` from `docs/site`. Pages live under `docs/site/pages` and are written in MDX so you can embed code blocks, callouts, or custom React components as needed.
